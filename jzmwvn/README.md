# Demo Usage Guide (Generation 2 Contest Example)

This demo uses CCXT to fetch Binance futures market data and queries account info and pushes target positions via the OMS client (`sdk/oms_client.py`). The core logic is in `demo.py`.

- Market data: ccxt.pro Binance futures 1m/1h K-lines
- Account & trading: OMS API via OmsClient
- Strategy framework: Strong/weak rotation executed daily at 08:00 (long top N, short bottom N), funds equally allocated

## Main Classes & Functions (`demo.py`)

- CryptoQuantDemo
  - available_symbols:
    - Tradable symbol list (e.g. BTC-USDT-PERP, ETH-USDT-PERP, ...)
  - get_historical_data():
    - Uses ccxt.pro (Binance REST) to fetch past 7 days of 1h K-lines, saves as DataFrame
  - on_1min_kline(symbol, kline):
    - 1-minute K-line callback; triggers run_strategy() at 08:00
  - get_account_balance():
    - Gets balance via OmsClient.get_balance(), converts to {asset: balance} dict
  - get_current_positions():
    - Gets positions via OmsClient.get_position()
  - push_target_positions(positions):
    - Pushes target positions to OMS. `positions` is {symbol: target value (USDT)}, positive for long, negative for short
    - Converts BTC-USDT-PERP to OMS-required instrument_name: BTC-USDT
    - instrument_type is always "future"
    - position_side mapped LONG/SHORT by sign, target_value is absolute value formatted to two decimals
  - calculate_7d_returns():
    - Calculates 7-day returns based on fetched 1h K-lines
  - run_strategy():
    - Strong/weak rotation:
      1) Calculate 7-day returns for all symbols
      2) Select top N (long) and bottom N (short), N=position_switch_count (default 2)
      3) Allocate 90% of account USDT balance equally to these long/short symbols
      4) Build target positions and call push_target_positions()
  - _run() / run() / cleanup():
    - Real-time 1m K-line subscription (ccxt.pro), triggers callbacks and periodic account refresh; Ctrl+C for graceful exit

## Tuning & Notes
- position_switch_count defaults to 2, adjust based on tradable symbols and risk control
- Tradable symbols are built-in via available_symbols, can be extended or dynamically fetched via OmsClient.get_symbols()
- Consider trading slippage and fees; recommend reserving safety margin in fund allocation and target position push
- For network/API errors, add retry and alert logic
- Avoid hardcoding OMS_URL and OMS_ACCESS_TOKEN in production

## Common Issues
- ccxtpro not installed or license issues: Check ccxtpro installation and authorization
- OMS authentication failed: Check OMS_ACCESS_TOKEN
- Historical data fetch empty: Check if symbol is tradable, or relax rate limits/delays
- Target position set failed: Check instrument_name, instrument_type, permissions, and risk limits
